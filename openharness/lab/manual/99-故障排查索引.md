## A · 安装与 CLI

### A1 CLI 找不到

**现象**：`'openh' 不是内部或外部命令`；lab 脚本 `[跳过]`

**排查**：

1. `uv sync --group openharness-cli`（在 `learn/`）
2. `uv run openh --version`
3. Windows 勿用 `oh`（PowerShell 别名冲突）→ 用 **`openh`**
4. `where.exe openh` / `Get-Command openh`

**详见**：[01-安装与配置.md](01-安装与配置.md)

---

### A2 setup 失败

**现象**：`openh setup` 报错、profile 未写入

**排查**：

1. 网络能否访问 Provider / Gateway
2. API Key 格式是否正确（无多余空格）
3. 检查 `%USERPROFILE%\.openharness\profiles\` 是否生成文件
4. 重跑 setup 或手动编辑 profile json

**详见**：[01-安装与配置.md](01-安装与配置.md) · [05-Provider与Profile.md](05-Provider与Profile.md)

---

### A3 dry-run blocked

**现象**：`readiness: blocked`，有 `next_actions`

**排查**（按 next_actions 顺序）：

1. 是否完成 setup / 有默认 profile
2. API Key 是否配置
3. MCP servers.json 路径、command 是否有效 → [08-MCP与Plugin.md](08-MCP与Plugin.md)
4. Skills 目录结构是否每 skill 含 SKILL.md

**命令**：

```bash
openh --dry-run
openh --dry-run --output-format json
uv run python openharness/stage-1/lab/scripts/02_run_dry_run.py
```

**详见**：[04-dry-run与readiness.md](04-dry-run与readiness.md)

---

### A4 dry-run warning

**现象**：能跑但有 warnings

**排查**：逐条读 warning；常见为 MCP 可选 server 未启、Skills 路径不存在、profile label 重复。staging 前尽量清零 warning。

---

### A5 非交互 `-p` 失败

**现象**：`returncode != 0`，stderr 有错误

**排查**：

1. 先 `openh --dry-run` 是否 ready
2. profile 是否正确、Key 是否有效
3. prompt 是否过长触发限流
4. 增大 timeout（脚本默认 120–180s）

**命令**：

```bash
openh -p "Reply pong" --output-format json
```

**详见**：[06-输出格式与集成.md](06-输出格式与集成.md)

---

### A6 API 密钥与 profile

**现象**：401、403、`invalid_api_key`、模型不存在

**排查**：

1. `openh provider list` → 当前 active profile
2. 该 profile 的 base_url、model、key 与公司 Gateway 文档一致
3. 切换 profile 后**再跑同一条** `-p` 对比
4. 公司 Gateway 是否需要 VPN / IP 白名单

**详见**：[05-Provider与Profile.md](05-Provider与Profile.md)

---

### A7 stream-json 问题

**现象**：无输出、只有 stderr、JSON 行不完整

**排查**：

1. 同 prompt 先试 `--output-format json` 确认 API 通
2. 分离 stderr：`openh ... 2>err.log`
3. 检查是否 timeout 过短
4. 保存原始输出到 `experiments/` 对照 [06-输出格式与集成.md](06-输出格式与集成.md) 事件类型

**脚本**：`stage-1/lab/scripts/03_run_stream_json.py`

---

### A8 TUI 问题

**现象**：TUI 黑屏、无法输入、立即退出

**排查**：

1. 终端是否支持交互（非 CI 无 TTY）
2. `--dry-run` 是否 blocked
3. 换 Windows Terminal / 非嵌套 IDE 终端试
4. 查 `--help` 是否有 `--no-tui` 类 debug  flag

**详见**：[03-TUI与斜杠命令.md](03-TUI与斜杠命令.md)

---

## B · Skills 与权限

### B1 Skills 不显示

**现象**：`/skills` 空或缺少 project skills

**排查**：

1. cwd 是否在含 `.openharness/skills/` 的目录（如 `stage-2/lab`）
2. 运行 `uv run python openharness/stage-2/lab/scripts/01_install_skills.py`
3. 每 skill 目录下是否有 `SKILL.md`
4. `allow_project_skills` 是否被设为 false

**详见**：[07-Skills与权限.md](07-Skills与权限.md)

---

### B2 Skill 不触发

**现象**：Skill 在列表中但模型不按流程走

**排查**：

1. SKILL.md 描述是否写清「何时使用」
2. 用 `/skill-name` 显式调用
3. 换 `/plan` 模式试结构化任务

---

### B3 权限异常

**现象**：该拦的没拦 / 不该弹窗一直弹

**排查**：

1. 查 `settings.json` 中 `permission.mode`（生产勿用 auto）
2. `path_rules` pattern 是否覆盖目标路径
3. 项目级 vs 用户级 settings 覆盖关系

**详见**：[07-Skills与权限.md](07-Skills与权限.md)

---

## C · MCP

### C1 MCP 配置错误

**现象**：dry-run MCP 段 blocked，next action 指向 servers

**排查**：

1. `uv run python openharness/stage-3/lab/scripts/02_print_mcp_config_example.py`
2. JSON 语法 valid
3. `command` 在 shell 中可执行（`uv`、`python` 在 PATH）
4. args 中路径相对 **启动 openh 时的 cwd**（建议 `cd learn`）

**详见**：[08-MCP与Plugin.md](08-MCP与Plugin.md)

---

### C2 MCP 启动超时

**现象**：server 启动慢、dry-run 或首次 tool 超时

**排查**：

1. 单独运行 server：`uv run python openharness/showcase/03_mcp_readonly_server/server.py --stdio`
2. 首次 `uv run` 是否在下包
3. 增大 Harness MCP timeout（若配置项存在）
4. server 内勿阻塞 stdin

---

### C3 MCP Tool 不可见

**现象**：dry-run 过但模型说不认识 tool

**排查**：

1. 重启 TUI 会话（改 config 后）
2. prompt 中 **点名** tool 名（如 `search_tickets`）
3. 确认 server 注册的工具名与 prompt 一致

---

## D · Memory 与 Agent 行为

### D1 MEMORY 不生效

**现象**：MEMORY.md 写了但 Agent 不知道

**排查**：

1. 文件路径：`.openharness/MEMORY.md` 在**项目 cwd** 下
2. 用 `01_seed_memory_md.py` 生成到 `stage-4/lab/.openharness/`
3. **新开会话**或 `/resume` 后再问
4. 问题是否具体指向 MEMORY 中已有事实

**详见**：[09-Memory与Multi-Agent.md](09-Memory与Multi-Agent.md)

---

### D2 Agent 死循环

**现象**：反复 tool_use、不给出最终答案

**排查**：

1. 加 `--max-turns 5` 限制
2. 简化 prompt、减少开放目标
3. 检查 Tool 是否总返回 error 导致模型重试
4. 阅读 engine loop：[06-engine-reading workbook](../../stage-1/lab/workbook/06-engine-reading.md)

---

### D3 max-turns

**现象**：设置 max-turns 仍超过轮数

**排查**：

1. 确认 CLI 版本支持该 flag（`openh --help`）
2. 用 `stage-4/lab/scripts/02_max_turns_prompt.py` 复现并保存 experiments
3. 区分「turn」与「token」限制

---

## E · Gateway 与集成

### E1 Gateway / Webhook

**现象**：curl 4xx/5xx、连接拒绝

**排查**：

1. showcase 服务是否启动（见 [10-Gateway与程序化API.md](10-Gateway与程序化API.md)）
2. 端口、路径、Content-Type
3. Gateway 日志中 Harness subprocess 是否 spawn 失败

---

### E2 JSON 解析

**现象**：集成代码取不到字段

**排查**：

1. 保存原始 stdout 到文件
2. 对比不同 openharness-ai 版本的 JSON schema
3. 非 0 exit 时勿只解析 stdout，读 stderr

---

## F · 本仓库 Lab 特有问题

### F1 lab 脚本 import 错误

**现象**：`ModuleNotFoundError: No module named '_shared'`

**修复**：从 **`learn/` 根**执行，勿改 `ROOT = parents[3]`：

```bash
cd learn
uv run python openharness/stage-1/lab/scripts/01_check_install.py
```

---

### F2 lab 脚本跳过

**现象**：输出 `[跳过]` + 安装提示

**修复**：

```bash
cd learn
uv sync --group openharness-cli
openh --version
```

---

### F3 Windows 编码

**现象**：PowerShell 中文乱码、`UnicodeEncodeError`

**排查**：

1. `$OutputEncoding = [Console]::OutputEncoding = [Text.UTF8Encoding]::UTF8`
2. 实验记录用 UTF-8 写 md；CLI 输出以 `experiments/*.json` 为准
3. lab 脚本成功标记为 `[OK]`（ASCII）

---

### F4 compose 路径

**现象**：`01_compose_checklist.py` 未找到 docker-compose.yml

**排查**：

1. 确认存在 `study/base/openharness/docker-compose.yml`
2. 脚本从 `learn` 根运行；`STUDY_ROOT = parents[5]` 指向 `study/`

---

## 仍无法解决？

1. 保存：`openh --version`、`openh --dry-run --output-format json`、相关 `experiments/*.json`
2. 对照官方文档与 [OpenHarness GitHub Issues](https://github.com/HKUDS/OpenHarness/issues)
3. 回到对应 stage lab workbook 补全观察题（Often 能反推 config 问题）
