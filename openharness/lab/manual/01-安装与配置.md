## 1.1 安装方式


| 方式     | 命令                                            | 适用         |
| ------ | --------------------------------------------- | ---------- |
| uv（推荐） | `cd learn && uv sync --group openharness-cli` | 本仓库 lab    |
| pip    | `pip install openharness-ai`                  | 全局或独立 venv |
| 升级     | `pip install -U openharness-ai`               | 版本过旧时      |


安装后验证：

```bash
openh --version    # Windows PowerShell
oh --version       # macOS / Linux
openh --help
```

---

## 1.2 Windows：`openh` 与 `oh`

PowerShell 中 `**oh` 常为 `Out-Host` 别名**，会吞掉真实 CLI。本仓库一律使用 `**openh`**。

若仍报错「找不到命令」：

```powershell
Get-Command openh
where.exe openh
# 确认 .venv\Scripts 或 pip 安装目录在 PATH 中
uv run openh --version
```

---

## 1.3 首次初始化 `setup`

```bash
openh setup
```

按向导完成：

1. 选择 Provider（OpenAI / Anthropic / 兼容 OpenAI 的 Gateway 等）
2. 填写 API Key 或 Base URL + Key
3. 命名 **profile**（建议：`dev` / `staging` / `prod-gw` 分离）

非交互环境可先配环境变量（见 `[.env.example](../../.env.example)`）：

```bash
# learn/openharness/.env.example
OPENAI_API_KEY=sk-...
```

## 1.4 配置目录结构

默认用户级配置（Windows 示例）：

```
%USERPROFILE%\.openharness\
├── settings.json          # 全局 settings（permission、默认 profile 等）
├── profiles/              # 各 Provider profile
└── mcp/                   # MCP servers 配置（版本可能略有差异）
```

项目级配置（在**当前工作目录**或项目根）：

```
your-project/
└── .openharness/
    ├── skills/            # 项目 Skills（SKILL.md 目录）
    ├── MEMORY.md          # 跨会话记忆（stage-4 lab）
    └── settings.json      # 项目覆盖（可选）
```

**原则**：公司 Skills / MCP 放 Git；密钥只进 profile 或 Secret，不进仓库。
