## 6.1 三种输出模式

| 模式 | 命令 flag | 输出形态 | 适用 |
|------|-----------|----------|------|
| 文本 | 默认 / TUI | 人类可读 | 本地调试 |
| JSON | `--output-format json` | 单次完整 JSON | REST 同步 API |
| stream-json | `--output-format stream-json` | NDJSON 事件流 | SSE、IM 机器人、管道 |

## 6.2 JSON 非交互

```bash
openh -p "Reply with one word: pong" --output-format json
```

stage-5 lab：

```bash
uv run python openharness/stage-5/lab/scripts/01_noninteractive_json.py
```

**集成要点**：

- 解析 `content` / `messages` / `stop_reason`（字段名以实际为准）
- 检查 `returncode`；非 0 时读 stderr
- 超时：长任务增大客户端 timeout（脚本默认 120s）

## 6.3 stream-json

```bash
openh -p "List three files in the current directory" --output-format stream-json
```

或使用仓库冒烟 prompt：

```bash
cd learn
uv run python openharness/stage-1/lab/scripts/03_run_stream_json.py
# 读取 stage-1/lab/prompts/smoke-test.txt
```

**常见事件类型**（名称因版本而异）：

- 文本增量：`content_block_delta` / `message_delta`
- Tool 调用：`tool_use` / `tool_call`
- 结束：`message_stop` / `result`

**练习**：保存原始 NDJSON 到 `experiments/stream-json-raw.txt`，标出至少 3 种事件行。

---

## 6.4 json vs stream-json 选型

| 场景 | 推荐 |
|------|------|
| 内部微服务同步调用 | json |
| Slack / 飞书 / WebSocket 推流 | stream-json |
| CI 断言单句回复 | json |
| 展示 Tool 调用过程 | stream-json |

## 6.5 简易 NDJSON 解析器（Python 思路）

```python
import json
import sys

for line in sys.stdin:
    line = line.strip()
    if not line:
        continue
    event = json.loads(line)
    print(event.get("type"), event.keys())
```

管道示例：

```bash
openh -p "..." --output-format stream-json 2>stderr.log | python parser.py
```
