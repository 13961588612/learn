## 11.3 发布 checklist

运行交互清单：

```bash
uv run python openharness/stage-7/lab/scripts/01_release_checklist.py
```

**标准项**（逐项打勾）：

```
[ ] openh --dry-run 全绿（含 MCP）
[ ] staging 冒烟：stream-json 一条完整会话
[ ] 审计日志采样 review（含 tool 链）
[ ] settings.json permission 与 prod 一致
[ ] Skills tag 与 deploy 版本一致
[ ] 密钥仅 Secret / .env，未进镜像
[ ] 回滚方案：上一版 image tag + Skills tag
[ ] 健康检查端点 + MCP 探活
```

## 11.4 健康检查

| 组件 | 检查方式 |
|------|----------|
| Gateway | `GET /health` → 200 |
| Harness | `openh --dry-run` in init container |
| MCP | dry-run MCP 段 + 独立 `stdio` ping |
| Postgres/Redis | compose healthcheck |

概念脚本：`stage-7/01_health_probe.py`

---

## 11.5 故障树（SRE 速览）

```
用户报告 Agent 无响应
├── Gateway 5xx → 查 Gateway 日志 / 上游 Harness
├── Harness timeout → profile / 模型 Gateway / max-turns
├── MCP 超时 → server 进程、网络、dry-run
├── 401 → profile key 轮换
└── 权限全拒 → settings permission mode / path_rules
```

详细条目见 [99-故障排查索引.md](99-故障排查索引.md)。
